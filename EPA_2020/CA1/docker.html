Dockerfile introduction

A Dockerfile is a text document that contains all the commands a user could call on the command line to assemble an image. Using docker build users can create an automated build that executes several command-line instructions in succession.

Step 1: Pre-Dockerfile creation

Make sure you are in the correct directory that you want to work in. To check your current directory use the command pwd.

If you are in the correct directory it is time to start creating the Dockerfile and the corresponding files.

Step 2: Creation of Dockerfile

Once we are in the correct directory it is time to create the Dockerfile. Use the command nano Dockerfile to create the Dockerfile. Once the Dockerfile is created there are a number of commands you can include in the Dockerfile. I used the following docker commands inside my Dockerfile:

FROM python:3 - This command creates a layer from the python3 docker image.

WORKDIR ./ - The WORKDIR command is used to move to the directory where your file is stored.

COPY dockerfile.html ./ - The copy command is used to bring your file to your repository. In my case, the file that I am using is called docker.html. It will pull this file from my working directory to my repositary.

CMD python -m http.server 8080 - This provides a list of commands to run when the docker image is being run. In this case, I am telling this to run the python3 server module on port 8080.

Step 3: Running Dockerfile

Before we run the Dockerfile we must first make sure that we have created our html file. In my case I sed the following command nano docker.html

Once you have created the html file and Dockerfile you can start to build the Dockerfile. To do this we use the docker build.. This command will run all of the commands inside the Dockerfile and once that is completed an image id will be created

Afterwards we use the following command to run the docker image docker run -d -p 8080:8080 "image id". The -d is detached mode, enables it to run in the background. The -p is used to bind the ports, in my case ports 8080:8080.

Once the docker image is running we will then go into web preview choose whatever port you have chosen, in my case it is port 8080. Once that is done you click on your html file and your docker image should appear.

Congrats, you have learned how to use docker.
